Cadastre seus produtos na Rappi.

Requisitos para criação de produtos

Para publicar seus produtos na Rappi, existem alguns pontos importantes:

  • Autenticação
  • Categorias de Produtos
  • Variações

Autenticação

Rappi utiliza o Bearer Token gerada no endpoint de login para autenticar as requisições. Para obter o token, você precisa realizar a autenticação no endpoint de login e obter o token que será utilizado no header em todos endpoints posteriores.

Categorias de Produtos

Elas são utilizadas no aplicativo para proporcionar ao usuário uma boa experiência de navegação e facilitar a busca pelo produto desejado. Pense nos corredores e prateleiras de um supermercado ou nos setores de roupas de uma loja de departamentos. Para criar um produto, é obrigatório definir categorias. Ela determinará quais atributos o produto deve possuir para pertencer àquela categoria.

Variações

Variações são utilizadas principalmentes para mesmos itens com características diferentes, como cor, tamanho, etc.

Endpoints

POST /login

Providencia o bearer de autenticação para as requisições subsequentes.

Requisição (cURL) 
curl --location 'https://services.rappi.com.br/api/open-api/login' 
                    --data '{
                      "client_id": "Solicitar ao time Rappi",
                      "client_secret": "Solicitar ao time Rappi"
                    }'

GET /categories

Lista as categorias de nível 1, o topo da árvore. Combine com category-tree para descer até a mais específica.

Requisição (cURL) 
curl --location 'https://services.rappi.com.br/api/open-api/v1/catalog/categories' \
--header 'Authorization: Bearer {bearer_token}'
Exemplo de resposta (JSON) 
{
  "data": [
    { "id": 3569, "name": "Acessórios bebês e crianças" },
    { "id": 2900, "name": "Animais de estimação" },
    { "id": 4190, "name": "Arte e Trabalhos manuais" }
  ]
}

GET /stores

Lista todas as lojas Rappi do parceiro com id, nome, endereço e coordenadas. Mapeie cada id ao código da loja no seu sistema antes de enviar estoque ou preço (API de inventário).

Sem corpo de requisição. Suporta paginação via limit e offset — itere até a resposta vir vazia para garantir cobertura total.

Requisição (cURL) 
curl --location 'https:services.rappi.com.br/api/open-api/v1/catalog/stores
--header 'Authorization: Bearer {bearer_token}'
Exemplo de resposta (JSON) 
{
  "data": [
    {
      "id": 1234,
      "country": "BR",
      "status": "published",
      "name": "Nome da loja",
      "lat": "4.678318",
      "long": "-74.05496",
      "address": "R. rappi, 123, Cidade, Estado, CEP, País"
    }
  ]
}

📘 Configurações de loja (endereço, horários, etc.) são gerenciadas pela equipe KAM da Rappi.

POST /products

Cria um produto na Rappi a nivel de marca.

  • Sem variação: 1 SKU → 1 id.
  • Com variação: defina has_variation: true. Cada item de product[] recebe um id próprio e todos compartilham o mesmo parent_id.

Campos gerais

  • category_id — id da categoria folha(categoria mais específica).
  • name — nome exibido na navegação e na busca.
  • description — descrição exibida na página de detalhes.
  • has_variation — false para item unitário, true para múltiplas variações.

Presentation

  • quantity — quantidade contida em uma unidade.
  • unitType — unidade de medida: "Und", "Kg", "g", "mg", "L", "mL", "m" ou "cm".

Exemplos de apresentação de produtos:

Sell type

  • type — forma de venda: "U" (unitário), "WB" (peso estimado por unidade) ou "WW" (peso fracionado).
  • minQuantity — quantidade mínima por compra (em "WB", a estimativa de peso por unidade).
  • maxQuantity — quantidade máxima por compra.
  • stepQuantity — incremento por adição (ex.: 100g só permite somar de 100 em 100g).

Exemplos de apresentação de produtos:

Product[]

  • sku — código interno do retailer.
  • ean — código de barras válido (8, 12, 13 ou 14 dígitos) ou null para produtos caseiros.
  • images — array de objetos { path, position } com URL pública HTTPS.
  • attributes — pares code: valor alinhados aos atributos da categoria folha.
Requisição (cURL — produto sem variação) 
curl --location 'https://services.rappi.com.br/api/open-api/v1/catalog/products' \
--header 'Authorization: Bearer {bearer_token}' \
--data-raw '{
  "category_id": 13,
  "name": "Rappi Shoes",
  "description": "Rappi Shoes",
  "has_variation": false,
  "presentation": { "quantity": 1, "unitType": "Und" },
  "sellType": { "type": "U", "minQuantity": 1, "maxQuantity": 10, "stepQuantity": 1 },
  "product": [
    {
      "sku": "1335",
      "ean": 7702122343494,
      "name": "Rappi Shoes",
      "images": [
        { "path": "https://rappi.com/product_0.jpg", "position": 1 },
        { "path": "https://rappi.com/product_1.jpg", "position": 2 }
      ],
      "attributes": { "color_alt": "white" }
    }
  ]
}'
Resposta (sem variação) 
{
  "category_id": 13,
  "name": "Rappi Shoes",
  "has_variation": false,
  "product": {
    "successful": [
      {
        "id": 2147484292,
        "parent_id": 151139,
        "sku": "1335",
        "ean": 730170648596,
        "images": [
          { "path": "https://rappi.com/product_0.jpg", "position": 1 }
        ],
        "attributes": { "color_alt": "white" }
      }
    ]
  }
}

Exemplo com variações

Para itens com variações (cor, tamanho, memória, etc.), envie múltiplos objetos em product[] com has_variation: true. Cada variação recebe um id próprio e todas compartilham o mesmo parent_id — é esse vínculo que permite ao app exibi-las agrupadas na página do produto.

Corpo da requisição (com variações) 
{
  "category_id": 12,
  "name": "Rappi Shirt",
  "description": "Rappi Really Cool Shirt",
  "has_variation": true,
  "presentation": { "quantity": 1, "unitType": "Und" },
  "sellType": { "type": "U", "minQuantity": 1, "maxQuantity": 10, "stepQuantity": 1 },
  "product": [
    { "sku": "NSD1235", "ean": "7730197002190", "attributes": { "color": "blue" } },
    { "sku": "NSD1236", "ean": "7730197002191", "attributes": { "color": "green" } }
  ]
}
Resposta (com variações) 
{
  "product": {
    "successful": [
      {
        "id": 2147484329,
        "parent_id": 151192,
        "sku": "NSD1235",
        "ean": "7730197002190",
        "attributes": { "color": "blue" }
      },
      {
        "id": 2147484330,
        "parent_id": 151192,
        "sku": "NSD1236",
        "ean": "7730197002191",
        "attributes": { "color": "green" }
      }
    ]
  }
}

📘 Imagens: PNG ou JPG, URL estática e pública via HTTPS, idealmente abaixo de 1 MB. ✅ Válido: https://example.com/images/product.png. ❌ Inválido: /index.php?option=imagens_capa&barras=....

🚧 Não é possível adicionar variação a um produto já criado. A funcionalidade virá via parent_id.

      "name": "Refresco Open",

GET /products/sku?skus=SKU1,SKU2

Busca em lote pelos SKUs internos. Limite de 200 SKUs por chamada, separados por vírgula (padrão para todos os países). Retorna um resumo; para os campos completos, use GET /products.

Requisição (cURL) 
curl --location 'https://services.rappi.com.br/api/open-api/v1/catalog/products/sku?skus=Rappi13350,Rappi13351' \
--header 'Authorization: Bearer {bearer_token}'
Exemplo de resposta (JSON) 
[

    "id": 905397117,
    "sku": "13350",
    "name": "Rappi Shoes",
    "longDescription": "Rappi Shoes",
    "productId": 123,

    "country": "BR"
  }
]

Domínios das imagens por país

Os endpoints retornam apenas o path da imagem. Para montar a URL final, prefixe com o domínio do país correspondente:

  • ARG — https://images.rappi.com.ar/products/
  • BRA — https://images.rappi.com.br/products/
  • CHL — https://images.rappi.cl/products/
  • COL — https://images.rappi.com/products/
  • CRI — https://images.rappi.com.cr/products/
  • ECU — https://images.rappi.com.ec/products/
  • MEX — https://images.rappi.com.mx/products/
  • PER — https://images.rappi.pe/products/
  • URY — https://images.rappi.com.uy/products/

Notas Importantes

Criando um produto do zero

  1. Categorias são obrigatórias para criar um produto. Utilize sempre o id mais específico (folha) para criar o produto.

  2. Se o produto não tiver ean, utilize 'null'.

  3. Garanta que o tipo de unidade seja adequado ao tipo do produto - Pesáveis x Unidades.
