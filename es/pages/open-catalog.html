Open Catalog

Registra tus productos en Rappi.

Requisitos para la creación de productos

Para publicar tus productos en Rappi, hay algunos puntos importantes:

  • Autenticación
  • Categorías de productos
  • Variaciones

Autenticación

Rappi utiliza el Bearer Token generado en el endpoint de login para autenticar las solicitudes. Para obtener el token, debes autenticarte en el endpoint de login y usar ese token en el encabezado Authorization en todas las solicitudes siguientes.

Categorías de productos

Se usan en la aplicación para ofrecer al usuario una buena experiencia de navegación y facilitar la búsqueda del producto deseado. Piensa en los pasillos y estanterías de un supermercado o en las secciones de ropa de una tienda por departamentos. Para crear un producto, es obligatorio definir categorías. Esto determinará qué atributos el producto debe tener para pertenecer a esa categoría.

Variaciones

Las variaciones se usan principalmente para el mismo artículo con características diferentes, como color, talla, etc.

Flujo de creación de producto

Endpoints

POST /login

Proporciona el Bearer de autenticación para las solicitudes siguientes.

Solicitud (cURL) 
curl --location 'https://services.rappi.com.br/api/open-api/login' 
                    --data '{
                      "client_id": "Solicitar al equipo de Rappi",
                      "client_secret": "Solicitar al equipo de Rappi"
                    }'

GET /categories

Lista las categorías de nivel 1, la parte superior del árbol. Combínalo con category-tree para bajar hasta la más específica.

Solicitud (cURL) 
curl --location 'https://services.rappi.com.br/api/open-api/v1/catalog/categories' \
--header 'Authorization: Bearer {bearer_token}'
Ejemplo de respuesta (JSON) 
{
  "data": [
    { "id": 3569, "name": "Accesorios para bebés y niños" },
    { "id": 2900, "name": "Mascotas" },
    { "id": 4190, "name": "Arte y manualidades" }
  ]
}

GET /stores

Lista todas las tiendas de Rappi del socio con id, nombre, dirección y coordenadas. Mapea cada id al código de tienda en tu sistema antes de enviar inventario o precio (API de inventario).

Sin cuerpo de solicitud. Soporta paginación mediante limit y offset - itera hasta que la respuesta llegue vacía para garantizar cobertura total.

Solicitud (cURL) 
curl --location 'https:services.rappi.com.br/api/open-api/v1/catalog/stores
--header 'Authorization: Bearer {bearer_token}'
Ejemplo de respuesta (JSON) 
{
  "data": [
    {
      "id": 1234,
      "country": "BR",
      "status": "published",
      "name": "Nombre de la tienda",
      "lat": "4.678318",
      "long": "-74.05496",
      "address": "R. rappi, 123, Ciudad, Estado, Código postal, País"
    }
  ]
}

📘 Las configuraciones de tienda (dirección, horarios, etc.) son gestionadas por el equipo KAM de Rappi.

POST /products

Crea un producto en Rappi a nivel de marca.

  • Sin variación: 1 SKU → 1 id.
  • Con variación: define has_variation: true. Cada item de product[] recibe su propio id y todos comparten el mismo parent_id.

Campos generales

  • category_id — ID de la categoría hoja (la categoría más específica).
  • name — nombre mostrado en la navegación y en la búsqueda.
  • description — descripción mostrada en la página de detalle.
  • has_variation — false para artículo unitario, true para múltiples variaciones.

Presentation

  • quantity — cantidad contenida en una unidad.
  • unitType — unidad de medida: "Und", "Kg", "g", "mg", "L", "mL", "m" ou "cm".

Ejemplos de presentación de productos:

Sell type

  • type — forma de venta: "U" (unitario), "WB" (peso estimado por unidade) ou "WW" (peso fraccionado).
  • minQuantity — cantidad mínima por compra (en "WB", la estimación de peso por unidad).
  • maxQuantity — cantidad máxima por compra.
  • stepQuantity — incremento por adición (ex.: 100g solo permite sumar de 100 en 100g).

Ejemplos de presentación de productos:

Product[]

  • sku — código interno del retailer.
  • ean — código de barras válido (8, 12, 13 ou 14 dígitos) o null para productos caseros.
  • images — arreglo de objetos { path, position } con URL pública HTTPS.
  • attributes — pares code: valor alineados con los atributos de la categoría hoja.
Solicitud (cURL - producto sin variación) 
curl --location 'https://services.rappi.com.br/api/open-api/v1/catalog/products' \
--header 'Authorization: Bearer {bearer_token}' \
--data-raw '{
  "category_id": 13,
  "name": "Rappi Shoes",
  "description": "Rappi Shoes",
  "has_variation": false,
  "presentation": { "quantity": 1, "unitType": "Und" },
  "sellType": { "type": "U", "minQuantity": 1, "maxQuantity": 10, "stepQuantity": 1 },
  "product": [
    {
      "sku": "1335",
      "ean": 7702122343494,
      "name": "Rappi Shoes",
      "images": [
        { "path": "https://rappi.com/product_0.jpg", "position": 1 },
        { "path": "https://rappi.com/product_1.jpg", "position": 2 }
      ],
      "attributes": { "color_alt": "white" }
    }
  ]
}'
Respuesta (sin variación) 
{
  "category_id": 13,
  "name": "Rappi Shoes",
  "has_variation": false,
  "product": {
    "successful": [
      {
        "id": 2147484292,
        "parent_id": 151139,
        "sku": "1335",
        "ean": 730170648596,
        "images": [
          { "path": "https://rappi.com/product_0.jpg", "position": 1 }
        ],
        "attributes": { "color_alt": "white" }
      }
    ]
  }
}

Ejemplo con variaciones

Para artículos con variaciones (cor, tamanho, memória, etc.), envía múltiples objetos en product[] com has_variation: true. Cada variación recibe su propio id y todas comparten el mismo parent_id — este vínculo permite que la app las muestre agrupadas en la página del producto.

Cuerpo de la solicitud (con variaciones) 
{
  "category_id": 12,
  "name": "Rappi Shirt",
  "description": "Rappi Really Cool Shirt",
  "has_variation": true,
  "presentation": { "quantity": 1, "unitType": "Und" },
  "sellType": { "type": "U", "minQuantity": 1, "maxQuantity": 10, "stepQuantity": 1 },
  "product": [
    { "sku": "NSD1235", "ean": "7730197002190", "attributes": { "color": "blue" } },
    { "sku": "NSD1236", "ean": "7730197002191", "attributes": { "color": "green" } }
  ]
}
Respuesta (con variaciones) 
{
  "product": {
    "successful": [
      {
        "id": 2147484329,
        "parent_id": 151192,
        "sku": "NSD1235",
        "ean": "7730197002190",
        "attributes": { "color": "blue" }
      },
      {
        "id": 2147484330,
        "parent_id": 151192,
        "sku": "NSD1236",
        "ean": "7730197002191",
        "attributes": { "color": "green" }
      }
    ]
  }
}

📘 Imágenes: PNG ou JPG, URL estática y pública por HTTPS, idealmente por debajo de 1 MB. ✅ Válido: https://example.com/images/product.png. ❌ Inválido: /index.php?option=imagens_capa&barras=....

🚧 No es posible agregar una variación a un producto ya creado. La funcionalidad llegará via parent_id.

GET /products

Lista los productos ya creados para tu retailer. Útil para verificación y conciliación. Pagina con limit (predeterminado 20, máximo 50) e offset (predeterminado 0).

Solicitud (cURL) 
curl --location 'https://services.rappi.com.br/api/open-api/v1/catalog/products?limit=50&offset=0' \
--header 'Authorization: Bearer {bearer_token}'
Ejemplo de respuesta (JSON) 
{
  "data": [
    {
      "images": [{ "path": "0570b4e7-dc01-47b7-a47f-2d92ac1e6c12.png", "position": 1 }],
      "id": 2147653816,
      "ean": "2200000312075",
      "sku": "31201",
      "name": "Refresco Open",
      "longDescription": "Producto de prueba para la tienda de open orders",
      "productId": 151268,
      "status": "published",
      "country": "CO",
      "presentation": { "description": "1 und", "quantity": 1, "unitType": "Und" },

GET /products/sku?skus=SKU1,SKU2

Búsqueda masiva por SKUs internos. Límite de 200 SKUs por llamada, separados por coma (estándar para todos los países). Devuelve un resumen; para los campos completos, usa GET /products.

Solicitud (cURL) 
curl --location 'https://services.rappi.com.br/api/open-api/v1/catalog/products/sku?skus=Rappi13350,Rappi13351' \
--header 'Authorization: Bearer {bearer_token}'
Ejemplo de respuesta (JSON) 
[
  {
    "images": [{ "path": "https://files.readme.io/...", "position": 1 }],
    "id": 905397117,
    "sku": "13350",
    "name": "Rappi Shoes",
    "longDescription": "Rappi Shoes",

    "status": "published",
    "country": "BR"
  }
]

Dominios de imágenes por país

Los endpoints devuelven solo el path de la imagen. Para construir la URL final, agrega como prefijo el dominio del país correspondiente:

  • ARG — https://images.rappi.com.ar/products/
  • BRA — https://images.rappi.com.br/products/
  • CHL — https://images.rappi.cl/products/
  • COL — https://images.rappi.com/products/
  • CRI — https://images.rappi.com.cr/products/
  • ECU — https://images.rappi.com.ec/products/
  • MEX — https://images.rappi.com.mx/products/
  • PER — https://images.rappi.pe/products/
  • URY — https://images.rappi.com.uy/products/

Notas importantes

Creando un producto desde cero

  1. Las categorías son obligatorias para crear un producto. Usa siempre el id más específico (hoja) para crear el producto.

  2. Si el producto no tiene EAN, usa null.

  3. Asegúrate de que el tipo de unidad sea adecuado al tipo de producto - pesables frente a unidades vendidas por unidad.
