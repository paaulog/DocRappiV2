Complex Discounts

Guia de integração para promoções complexas sobre o inventário: layout de arquivos SFTP, tipos de formato (L/P, X/Y, P/U), regras de preço e desativação.

O que é a integração Complex Discounts?

Complex é a opção de integração em que são tratados descontos que não são simples. Até o momento, há três tipos de descontos ou bundles que podem ampliar e melhorar a proposta de vendas dos nossos parceiros.

Por que integrar com Complex Discounts?

  • Até 25% a mais de crescimento em vendas em comparação com quem não aplica descontos.
  • Mais tráfego de usuários, ao tornar as lojas mais atraentes.
  • Maior visibilidade: mais promoções, mais banners, notificações push e redes sociais.
Importante: a Rappi só dá suporte à criação, inserção e gestão de promoções complexas constituídas por um único SKU (produto). A combinação de SKUs diferentes não é suportada.

Tipos de Complex Discounts

Compre (L) e pague (P) (X%) de desconto na (Y)ª unidade (P)% de desconto a partir de (U) unidades
Lógica de bundle expressa como Lx / Py (ex.: compre 3, pague 2). Percentual de desconto em uma posição específica de unidade no bundle (ex.: segunda unidade). Percentual de desconto no preço unitário a partir de um limiar mínimo de quantidade.

Como a integração Complex Discounts funciona

A integração Complex Discount funciona apenas via SFTP. Se a integração de inventário for por SFTP, será criada uma subpasta chamada discounts. Se for por API, a pasta será compartilhada e os arquivos ficarão no mesmo local.

O formato do arquivo deve ser .csv. É possível enviar os descontos de todas as lojas em um único arquivo ou em arquivos distintos por loja.

Como os arquivos devem ser nomeados

  • Data + hora: STOREID_YYYY-MM-DD_HHmm.csv
  • Só data: STOREID_YYYY-MM-DD.csv
  • Escopo parceiro: PARTNERNAME_YYYY-MM-DD.csv
Exemplo: a loja 123 pode ser 123_2022-01-31_1537.csv ou 123_2022-01-31.csv.

Campos da integração

Nome da coluna Tipo Obrigatório Descrição
store_id string TRUE Número de identificação da loja do parceiro
start_date string TRUE Início da promoção AAAA/MM/DD
end_date string TRUE Fim da promoção AAAA/MM/DD
description string TRUE Descrição da promoção. Exemplo: "Compre 3, Pague 2"
type_format string TRUE Formato da promoção. Exemplo: B3_P2
name string TRUE Nome do produto

Formato de data: revise o formato de data no arquivo. Qualquer outro formato impedirá o processamento correto da integração. Use AAAA/MM/DD conforme documentado acima.

Tipos de formato

Tipo de formato Exemplo
LX_PY L3_P2 — compre 3, pague 2 unidades.
XP_YU X25_Y2 — 25% de desconto na segunda unidade.
PP_UU P20_U4 — a partir da 4ª unidade, o preço unitário tem 20% de desconto.

Exemplo de arquivo

CSV de exemplo (cabeçalho + uma linha) 
store_id,start_date,end_date,description,type_format,name,id
123,2022/01/31,2022/02/28,"Compre 3, Pague 2",B3_P2,Produto de exemplo,SKU-001

A considerar

  • Os descontos começam às 00:00 GMT+0 no dia indicado no arquivo.
  • Os descontos terminam às 23:59:59 GMT+0 no dia indicado no arquivo.
  • O formato de data no arquivo deve ser AAAA/MM/DD.
  • Todos os parâmetros são obrigatórios.

Considerações técnicas

Para ter integração Complex Discount, é necessário já possuir uma integração de inventário ativa e operando dentro dos parâmetros gerais considerados uma integração saudável.

Para controlo de disponibilidade, a Rappi usa a regra de que o SKU de unidade única na integração de inventário deve ter pelo menos três vezes (3×) o total de unidades do bundle para o produto ser exibido na aplicação. Por exemplo, se o parceiro enviar uma promoção com formato como L5_P4, o produto original deve ter mais de 15 unidades em stock no ficheiro de inventário; caso contrário, o bundle será desativado.

A Rappi calcula o preço com desconto do bundle conforme o tipo de formato da seguinte forma:

LX_PY XP_YU PP_UU
Preço Preço por unidade × x Preço por unidade × u Preço por unidade
Preço com desconto Preço por unidade × y Preço por unidade × ((u − 1) + (1 − p/100)) U = unidade a partir da qual se aplica o novo preço unitário (cada linha pode representar um degrau na profundidade do desconto).

Cada linha lógica pode representar um aumento de desconto (ou uma redução no preço unitário), conforme o formato.

Em todos os casos: desconto percentual = 100 × (Preço cheio − Preço com desconto) / Preço cheio (arredonde o resultado para duas casas decimais).

Desativação de descontos

Data de fim

Com base no valor enviado no campo end_date, a Rappi aplica uma rotina diária para desativar as promoções que terminaram.

Estoque de segurança

A quantidade do SKU do produto deve ser pelo menos 3× o total de unidades do bundle para aparecer como ativo na app:

  • Exemplo: um bundle "Compre 5, Pague 4" precisa de pelo menos 15 unidades (SKU de unidade única) em stock.
  • Exemplo: um bundle "50% de desconto na 2ª unidade" precisa de pelo menos 6 unidades (SKU de unidade única) em stock ou inventário.
