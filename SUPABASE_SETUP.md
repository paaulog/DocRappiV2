# Supabase Setup (Opcao 3)

Este projeto agora suporta edicao compartilhada de conteudo com login/senha via Supabase.

## 1) Criar projeto e autenticao

- Crie um projeto no [Supabase](https://supabase.com/).
- Em `Authentication > Providers`, habilite `Email`.
- Crie os usuarios editores em `Authentication > Users`.

## 2) Criar tabela de conteudo

Execute no SQL Editor:

```sql
create table if not exists public.docs_content (
  id bigint generated by default as identity primary key,
  language text not null,
  technology text not null,
  content_json jsonb not null,
  updated_by text,
  updated_at timestamptz not null default now(),
  constraint docs_content_unique unique (language, technology)
);

alter table public.docs_content enable row level security;
```

## 3) Politicas (RLS)

Use politicas simples para editores autenticados:

```sql
create policy "read docs for all authenticated"
on public.docs_content
for select
to authenticated
using (true);

create policy "upsert docs for all authenticated"
on public.docs_content
for all
to authenticated
using (true)
with check (true);
```

Se quiser leitura publica sem login, adicione politica para `anon` em `select`.

## 4) Configurar frontend

Em `index.html`, preencha:

```html
window.__APP_CONFIG = {
  SUPABASE_URL: 'https://SEU-PROJETO.supabase.co',
  SUPABASE_ANON_KEY: 'SUA_ANON_PUBLIC_KEY',
};
```

## 5) Fluxo de uso

- Usuario abre `Modo editor`.
- Faz login com email/senha do Supabase.
- Edita e clica `Salvar para todos`.
- Conteudo e salvo em `docs_content` e fica disponivel para os proximos acessos.

## Observacao de deploy (Netlify)

Como a app e estatica, basta publicar normalmente no Netlify.
As credenciais `SUPABASE_URL` e `SUPABASE_ANON_KEY` sao publicas (anon key) e podem ficar no frontend.
