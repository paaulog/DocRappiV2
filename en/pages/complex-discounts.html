Complex Discounts

Integration guide for complex promotions over inventory: SFTP file layout, format types (L/P, X/Y, P/U), pricing rules, and deactivation.

What is Complex Discounts Integration?

Complex is the integration option where discounts that are not simple are handled. So far, there are three types of discounts or bundles that can broaden and improve the sales proposition of our partners.

Why integrate with Complex Discounts?

  • Up to 25% sales growth compared to those who do not apply discounts.
  • Increased user traffic, by making stores more attractive to users.
  • Greater visibility: more promos, more banners, push notifications, and social media.
Important: Rappi only supports the creation, insertion, and management of complex promotions that consist of a single SKU (product). The combination of different SKUs is not supported.

Complex Discounts types

Buy (L) and pay (P) (X%) discount on the (Y) unit (P)% discount starting from (U) units
Bundle logic expressed as Lx / Py (e.g. buy 3, pay 2). Percentage off a specific unit position in the bundle (e.g. second unit). Percentage off unit price from a minimum quantity threshold onward.

How Complex Discounts integrates

The Complex Discount integration works only via SFTP. If your inventory integration is through SFTP, a subfolder called discounts will be created. If it is through API, the folder will be shared, and the files will be left in the same location.

The file format must be .csv. You can send the discounts for all stores in a single file or in different files per store.

How files should be named

  • Date + time: STOREID_YYYY-MM-DD_HHmm.csv
  • Date only: STOREID_YYYY-MM-DD.csv
  • Partner scope: PARTNERNAME_YYYY-MM-DD.csv
Example: Store 123 could be 123_2022-01-31_1537.csv or 123_2022-01-31.csv.

Integration fields

Column name Type Required Description
store_id string TRUE Partner store identification number
start_date string TRUE Promotion starts YYYY/MM/DD
end_date string TRUE Promotion ends YYYY/MM/DD
description string TRUE Promotion description. Example: "Buy 3, Pay 2"
type_format string TRUE Promotion format. Example: B3_P2
name string TRUE Product name
id string TRUE SKU used for general integration
Date format: review the date format in the file. Any other format will prevent the integration from being processed correctly. Use YYYY/MM/DD as documented above.

Format types

Type format Example
LX_PY L3_P2 — buy 3, pay for 2 units.
XP_YU X25_Y2 — 25% discount on the second unit.
PP_UU P20_U4 — from the 4th unit onward, the unit price has a 20% discount.

File example

Sample CSV (header + one row) 
store_id,start_date,end_date,description,type_format,name,id
123,2022/01/31,2022/02/28,"Buy 3, Pay 2",B3_P2,Example Product,SKU-001

To consider

  • Discounts start at 12:00 a.m. GMT+0 on the day specified in the file.
  • Discounts end at 11:59:59 p.m. GMT+0 on the day specified in the file.
  • The date format in the file must be YYYY/MM/DD.
  • All parameters are mandatory.

Technical considerations

To have a Complex Discount integration, you must already have an inventory integration that is active and operating within the general parameters considered a healthy integration.

For availability control, Rappi uses the rule that the single-unit SKU in the inventory integration must have at least three times (3×) the total units in the bundle for the product to be shown in the application. For example, if the partner sends a promotion with format like L5_P4, the original product must have more than 15 units in stock in the inventory file; otherwise the bundle will be deactivated.

Rappi calculates the bundle discount price from the format type as follows:

LX_PY XP_YU PP_UU
Price Price per unit × x Price per unit × u Price per unit
Discounted price Price per unit × y Price per unit × ((u − 1) + (1 − p/100)) U = unit from which the new price per unit applies (each row can represent a step in discount depth).

Each logical row can represent a discount increase (or a reduction in the unit price) depending on the format.

For all cases: percentage discount = 100 × (Full price − Discounted price) / Full price (round the result to two decimal places).

Discount deactivation

End date

Based on the value sent in the end_date field, Rappi applies a daily routine to deactivate promotions that have finished.

Security stock

The quantity for the product SKU must be at least 3× the total units in the bundle for it to appear as active in the app:

  • Example: a "Buy 5, Pay 4" bundle needs at least 15 units (single-unit SKU) in stock.
  • Example: a "50% discount on the 2nd unit" bundle needs at least 6 units (single-unit SKU) in stock or inventory.
