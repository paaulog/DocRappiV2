Open Catalog

Register your products on Rappi.

Requirements for creating products

To publish your products on Rappi, there are a few key points to keep in mind:

  • Authentication
  • Product Categories
  • Variations

Authentication

Rappi uses the Bearer Token generated by the login endpoint to authenticate requests. To get the token, authenticate via the login endpoint and use that token in the header Authorization in all subsequent requests.

Product Categories

Categories are used in the app to provide users with a better browsing experience and make it easier to find the desired product. Think of supermarket aisles and shelves, or clothing sections in a department store. To create a product, defining categories is mandatory. It determines which attributes the product must have to belong to that category.

Variations

Variations are mainly used for the same item with different characteristics, such as color, size, etc.

                    --data '{

GET /categories

Lists level 1 categories, the top of the tree. Combine with category-tree to drill down to the most specific one.

Request (cURL) 
curl --location 'https://services.rappi.com.br/api/open-api/v1/catalog/categories' \
--header 'Authorization: Bearer {bearer_token}'
Response example (JSON) 
{
  "data": [
    { "id": 3569, "name": "Baby and kids accessories" },
    { "id": 2900, "name": "Pets" },
    { "id": 4190, "name": "Arts and crafts" }
  ]
}

GET /categories/{category_id}

Category details, including the list of attributes. Always call it with the last-level category id (the 'code' property of each attribute that will be included in the creation payload).

No request body.

Request (cURL) 
curl --location 'https://services.rappi.com.br/api/open-api/v1/catalog/categories/3093' \
--header 'Authorization: Bearer {bearer_token}'
Response example (JSON) 
{
  "id": 3093,
  "name": "Sore throat",
  "attributes": [
    {
      "id": 9,
      "name": "Packaging type",
      "code": "packaging",
      "validation": { "type": "string" },
      "description": "Indicates the type of beverage packaging",
      "isRequired": false
    },
    {
      "id": 1058,
      "name": "Active ingredient",
      "code": "active_ingredient",
      "validation": { "type": "string" },
      "description": "The active ingredient in a medication...",
      "isRequired": false
    }

}

💡 Example hierarchy down to the leaf: Pharmacy > Flu-and-cough > Sore throat (3093).

GET /stores

Lists all partner Rappi stores with id, name, address, and coordinates. Map each id to the store code in your system before sending stock or price updates (inventory API).

No request body. Supports pagination using limit and offset - iterate until the response is empty to ensure full coverage.

Request (cURL) 
curl --location 'https:services.rappi.com.br/api/open-api/v1/catalog/stores
--header 'Authorization: Bearer {bearer_token}'
Response example (JSON) 
{
  "data": [
    {
      "id": 1234,
      "country": "BR",
      "status": "published",
      "name": "Store name",
      "lat": "4.678318",
      "long": "-74.05496",
      "address": "R. rappi, 123, City, State, ZIP, Country"
    }
  ]
}

📘 Store settings (address, business hours, etc.) are managed by Rappi's KAM team.

POST /products

Creates a product on Rappi at the brand level.

  • No variation: 1 SKU → 1 id.
  • With variation: set has_variation: true. Cada item de product[] gets its own id and all share the same parent_id.

General fields

  • category_id — leaf category ID (the most specific category).
  • name — name shown in navigation and search.
  • description — description shown on the details page.
  • has_variation — false for a single item, true para multiple variations.

Presentation

  • quantity — quantity contained in one unit.
  • unitType — unit of measure: "Und", "Kg", "g", "mg", "L", "mL", "m" ou "cm".

Product presentation examples:

Sell type

  • type — sale format: "U" (unit), "WB" (estimated weight por unidade) ou "WW" (fractional weight).
  • minQuantity — minimum quantity per purchase (in "WB", the estimate of weight per unit).
  • maxQuantity — maximum quantity per purchase.
  • stepQuantity — increment per addition (ex.: 100g only allows adding in 100 in 100g).

Product presentation examples:

Product[]

  • sku — retailer internal code.
  • ean — valid barcode (8, 12, 13 ou 14 dígitos) or null for homemade products.
  • images — array of objects { path, position } with public HTTPS URL.
  • attributes — code: value pairs aligned with leaf category attributes.
Request (cURL - product without variation) 
curl --location 'https://services.rappi.com.br/api/open-api/v1/catalog/products' \
--header 'Authorization: Bearer {bearer_token}' \
--data-raw '{
  "category_id": 13,
  "name": "Rappi Shoes",
  "description": "Rappi Shoes",
  "has_variation": false,
  "presentation": { "quantity": 1, "unitType": "Und" },
  "sellType": { "type": "U", "minQuantity": 1, "maxQuantity": 10, "stepQuantity": 1 },
  "product": [
    {
      "sku": "1335",
      "ean": 7702122343494,
      "name": "Rappi Shoes",
      "images": [
        { "path": "https://rappi.com/product_0.jpg", "position": 1 },
        { "path": "https://rappi.com/product_1.jpg", "position": 2 }
      ],
      "attributes": { "color_alt": "white" }
    }
  ]
}'
Response (without variation) 
{
  "category_id": 13,
  "name": "Rappi Shoes",
  "has_variation": false,
  "product": {
    "successful": [
      {
        "id": 2147484292,
        "parent_id": 151139,
        "sku": "1335",
        "ean": 730170648596,
        "images": [
          { "path": "https://rappi.com/product_0.jpg", "position": 1 }
        ],
        "attributes": { "color_alt": "white" }
      }
    ]
  }
}

Example with variations

For items with variations (cor, tamanho, memória, etc.), send multiple objects in product[] com has_variation: true. Each variation gets its own id and all share the same parent_id — this link allows the app to display them grouped on the product page.

Request body (with variations) 
{
  "category_id": 12,
  "name": "Rappi Shirt",
  "description": "Rappi Really Cool Shirt",
  "has_variation": true,
  "presentation": { "quantity": 1, "unitType": "Und" },
  "sellType": { "type": "U", "minQuantity": 1, "maxQuantity": 10, "stepQuantity": 1 },
  "product": [
    { "sku": "NSD1235", "ean": "7730197002190", "attributes": { "color": "blue" } },
    { "sku": "NSD1236", "ean": "7730197002191", "attributes": { "color": "green" } }
  ]
}
Response (with variations) 
{
  "product": {
    "successful": [
      {
        "id": 2147484329,
        "parent_id": 151192,
        "sku": "NSD1235",
        "ean": "7730197002190",
        "attributes": { "color": "blue" }
      },
      {
        "id": 2147484330,
        "parent_id": 151192,
        "sku": "NSD1236",
        "ean": "7730197002191",
        "attributes": { "color": "green" }
      }
    ]
  }
}

📘 Images: PNG ou JPG, static and public HTTPS URL, ideally under 1 MB. ✅ Valid: https://example.com/images/product.png. ❌ Invalid: /index.php?option=imagens_capa&barras=....

🚧 It is not possible to add a variation to a product already created. This feature will come via parent_id.

GET /products

Lists products already created for your retailer. Useful for checking and reconciliation. Paginate with limit (default 20, maximum 50) e offset (default 0).

Request (cURL) 
curl --location 'https://services.rappi.com.br/api/open-api/v1/catalog/products?limit=50&offset=0' \
--header 'Authorization: Bearer {bearer_token}'
Response example (JSON) 
{
  "data": [
    {
      "images": [{ "path": "0570b4e7-dc01-47b7-a47f-2d92ac1e6c12.png", "position": 1 }],
      "id": 2147653816,
      "ean": "2200000312075",
      "sku": "31201",
      "name": "Refresco Open",
      "longDescription": "Producto de prueba para la tienda de open orders",
      "productId": 151268,
      "status": "published",
      "country": "CO",
      "presentation": { "description": "1 und", "quantity": 1, "unitType": "Und" },

  "offset": 0

Batch lookup by internal SKUs. Limit of 200 SKUs per call, separated by comma (standard for all countries). Returns a summary; for complete fields, use GET /products.

Request (cURL)

--header 'Authorization: Bearer {bearer_token}'
Response example (JSON)

Image domains by country

Endpoints return only the image path. To build the final URL, prefix it with the corresponding country domain:

  • ARG — https://images.rappi.com.ar/products/
  • BRA — https://images.rappi.com.br/products/
  • CHL — https://images.rappi.cl/products/
  • COL — https://images.rappi.com/products/
  • CRI — https://images.rappi.com.cr/products/
  • ECU — https://images.rappi.com.ec/products/
  • MEX — https://images.rappi.com.mx/products/
  • PER — https://images.rappi.pe/products/
  • URY — https://images.rappi.com.uy/products/

Important notes

Creating a product from scratch

  1. Categories are required to create a product. Always use the most specific id (leaf) to create the product.

  2. If the product has no EAN, use null.

  3. Make sure the unit type fits the product type - weighted products versus units sold per unit.
